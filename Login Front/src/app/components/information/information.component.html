A tener en cuenta del lado del Cliente

      Inputs Auth Login
    1. Validators.required
    2. Validators.pattern(REGEX EMAIL)
    3. Bloqueo de usuario después de 5 intentos fallidos (1m)
      Show Values DOM
    1. Los datos provenientes de la API y que se muestre en el DOM, deben pasar por el pipe protect-xss.
    2. Al imprimir variables en el dom, no usar doble llave, sino el atributo [innerHTML]="variable | pipe"

    A tener en cuenta del lado del Servidor

      Json Web Token
    1. Utilizar tokens para el inicio de sesiones
    2. El token se debe almacenar en una tabla de sesiones con datos importantes: ip, estado de sesión, token, activo
    3. Los datos encriptados en el token deben ser mínimos: userID, user/email, roles/permisos, configuraciones personalizadas, fecha de expiración
    4. En cada consulta al servidor o ejecución del CRUD, se debe enviar y validar el token
    5. Lo mejor es almacenar el token en cookies, pero es opcional
    6. Evitar el robo de tokens validando también la ip de consulta
